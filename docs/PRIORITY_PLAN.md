# ARUGULA Priority Plan

## Execution rule
Run ARUGULA loops on **one prioritized project at a time** until the loop machinery is producing clean evidence. Then add the next project.

## Priority order
1. memU
2. rareagent.work
3. BuildBid
4. trading
5. Fermware

## Why memU first
- fastest path to replay-grade evidence
- shared memory quality affects every other project
- easier to evaluate retrieval usefulness before larger product workflows

## memU success criteria before expanding
- judged benchmark set exists
- first replay run completed
- scorecard produced with explicit sample counts
- one mutation target chosen on evidence
- rollback rule recorded

## Expansion gate
Do not add the next project until memU has at least:
- 2 clean replay loops
- 1 retained improvement or clear rejected hypothesis
- stable artifact path and scorecard pattern reusable by the next project
